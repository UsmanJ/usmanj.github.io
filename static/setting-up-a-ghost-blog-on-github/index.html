Setting up a Ghost blog on GitHub

Having struggled to host my blog on GitHub I thought I'd write up a blog post about it to help everyone who may be struggling with it too.

Installing Node.js

To begin with, we need to install Node.js because Ghost is written in Node.js.

To do this, open your terminal and enter the following command:

  • brew install node

This should install Node.js

Installing Buster

Now we need to install Buster which is a tool that will allow you to export your Ghost blog pages into static pages; which you can then deploy to your blog on GitHub. To install Buster we will also need to install Python.

To install Buster, enter the following commands in your terminal:

  • brew install python
  • brew install wget
  • pip install buster

This should install Buster for you.

Setting up Ghost locally

Now we need to set up Ghost locally. To do this you will need to download Ghost from their website (https://www.ghost.org).

Once you have downloaded ghost and exported it into the directory that you want to keep it in, you will need to set it up locally.

Start off by opening another terminal window and going to your ghost directory.

Install npm using the following command. npm is a manager for installing Node.js applications.

  • npm install

Now that we have installed npm, we need to start it in order to run Ghost locally. We do this by using the following command:

  • npm start

Now Ghost will be running locally on your machine.

Go to the following link in order to create a Ghost account locally.

  • http://127.0.0.1:2368/ghost

Create a GitHub repository

Once you have created your account, you will now need to create a new repository on GitHub which will be connected to your ghost blog. Leave ghost to run locally in the background and just minimise that shell for now as we will not be needing it (but will need it to carry on running locally in the background for when we want to deploy the blog to GitHub).

Go to your GitHub account and create a repository called "username.github.io". Change username to your own username. For example in my case the repository is called "usmanj.github.io".

Create a directory for your static pages

Now go back to your first terminal shell (which should be in the ghost directory) and create a new folder. Call it the same name as your repository (username.github.io). You do not have to do this, but I did this because it will allow you to keep your static pages separate.

  • mkdir username.github.io
  • cd username.github.io

Setup Buster

Now that you have made another directory for the static pages, you can setup Buster in that folder.

  • buster setup https://github.com/username/username.github.io.git

Once this is done, run the following command which will generate all the needed files under the static subfolder created in the previous step.

  • buster generate --domain=http://127.0.0.1:2368

Push static folder to GitHub

Now enter that directory in your terminal by using cd static and then add all the files in that static folder to your git. Initialise your git by git init followed by adding all files to the git by running git add --all. Once this is done, commit the changes using git commit -m "Initial commit" and finally push it all to your GitHub repository.

To push to your repository you will need to add the remote using the following commands (replace username with your own username):

  • git remote add origin https://github.com/username/username.github.io
  • git push -u origin master

HURRRAAAYYY!!! Your blog should now be online on username.github.io.

If it isn't, drop me a message on Slack and I will try to help you get it up and running.

Deploying changes to GitHub

Last but not least, how will we deploy any changes to the local blog?

Once you have added/edited/deleted blog posts from your local ghost blog, you will need to deploy the changes to GitHub. To do this, go to your username.github.io folder directory in terminal (do not go into static).

Once your are in there and your local ghost is running in another shell; run the following commands:

  • buster generate
  • buster deploy

Hopefully this helps you guys if you were stuck.

Good luck!
