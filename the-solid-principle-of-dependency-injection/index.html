The SOLID principle of 'dependency inversion'

The dependency inversion principle provides us with two recommendations:

  • High-level modules should not depend on low-level modules. Both should depend on abstractions.
  • Abstractions should not depend upon details. Details should depend upon abstractions.

In simple words, you want to be able to change parts of your code without having to make multiple changes. This would enhance the usability of the code as it could potentially be used in many different scenarios and can be modified with ease if need be.

Dependency injection is the method used to succeed in dependency inversion. The dependency injection method is when you inject dependencies into an object rather than having the object create its own dependencies. This way you get more flexibility, less coupling and also testing such code is much easier.

An example of dependency injection is below:

# a typical approach
class Hacker
def initialize
@keyboard = Keyboard.new(:layout => 'us')
end
end
# and now with DI
class Hacker
def initialize(keyboard)
@keyboard = keyboard
end
end
